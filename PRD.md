## 2. 系统核心逻辑与 UI 优化

### 2.1 训练流量监控系统 (Traffic Status)
- **展示位置**: 详情页“体检时间”段落下方。
- **采集字段**: `studentCount` (今日目前在学人数，支持后台/手动录入)。
- **逻辑规则**:
    | 人数区间 (N) | 状态标签 | 视觉反馈 (颜色) |
    | :--- | :--- | :--- |
    | N <= 25 | **流畅** | 翡翠绿 (Emerald) |
    | 30 < N <= 60 | **较拥挤** | 橙色 (Orange) |
    | 60 < N <= 80 | **拥堵** | 红色 (Red) |
    | N > 80 | **堵爆了** | 深红色 (Red-700) |
- **UI 呈现**: 包含人数文字提示、状态标签以及动态进度条（进度百分比随状态阶梯式变化）。

## 3. 技术实施注释
- **数据结构**: `Store` 接口新增 `coaches` 数组、`pricing` 数组、`medicalCheckupTime` 字符串及 `studentCount` 数字字段。
- **地图集成**: 所有位置点坐标采用 [GCJ-02] 火星坐标系。
- **组件规范**: 进度条采用 Tailwind CSS 动态宽度渲染，过渡动画使用 `motion/react`。
